Part 3 · 01 Software & Getting Ready
WPILib 2027

New software,
same toolbox.

SystemCore is a new board, but the way you write robot code barely changes. WPILib 2027 is one unified software experience, your familiar languages still work, and there is a brand-new Driver Station that finally runs on every kind of computer.

Alpha preview · this will change

SystemCore is still in alpha and beta testing. The details here are based on what FIRST and WPILib have shared so far and they are not final, so confirm anything important against the official sources on the course page before you rely on it.

One library, two programs, the same code structure you already know.

Think of WPILib like the toolbox you carry to every build. For 2027 the toolbox gets reorganized into one unified experience that covers both FRC and FTC, but the tools inside are the ones you already use. You still write command-based code, you still describe your robot in subsystems and commands, and you still target the same vendor motor controllers. The big difference is which board you deploy to.

Why "unified" matters

Today FRC and FTC use different software worlds, so a programmer moving between them has to relearn the tools. WPILib 2027 brings both under one roof. It is like learning to ride one bike and finding that the brakes and gears work the same on every other bike in the shed. What you learn on one program carries straight to the other.

The control room

A Driver Station for every computer

The Driver Station is the program your drive team runs on a laptop at the field. It reads your gamepads, talks to the robot, and shows you what is happening. The old one had a catch: it ran on Windows only. The new Driver Station, built specifically for SystemCore, runs on Windows, macOS, and Linux. The roboRIO keeps using the old one, so only SystemCore teams get the new app.

🎚️
16-bit axesJoystick movement is read far more finely, up from 8-bit, so steering feels smoother.
🔘
Up to 64 buttonsPlus 8 POVs (the d-pad hats), so big custom control boards are no problem.
📳
Rumble + LEDThe app can buzz a controller or light its LEDs to signal the driver.
📐
Familiar layoutStill the docked strip about 200 pixels tall, so it looks like home.
Still in development

The new Driver Station is not finished. It is currently closed-source, and competition-field support (the FMS link that runs official matches) is still being added. Popular dashboards like Elastic are being updated to connect to it. Expect the look and the feature list to keep shifting until it is final.

Your code structure does not change

This is the part to relax about. The way you organize a robot project, subsystems, commands, the main robot class, is the same as what this academy already teaches. You point the same code at SystemCore instead of the roboRIO. The skills transfer.

Going further · optional

16-bit axis resolution means each joystick reading lands in one of roughly 65,000 steps instead of 256. For a slow, precise approach to a game piece, that finer reading lets your code tell tiny stick nudges apart that the old 8-bit reading would have lumped together. Here is one to think through with a mentor: where on your robot would smoother, finer joystick control actually change how the driver plays, and where would it make no difference at all?

Quick check

Pick an answer to see why.

Question 1
What is new about the SystemCore Driver Station compared to the old one?
The old Driver Station was Windows-only. The new one, built for SystemCore, runs on Windows, macOS, and Linux. The roboRIO keeps the existing Driver Station, so only SystemCore uses the new app.
Question 2
Which text languages does WPILib 2027 officially support on SystemCore?
The three officially supported text languages are Java, C++, and Python (Python via RobotPy), and all of them run on SystemCore. There is also Blockly for block-based coding.
Question 3
Which of these is a real new feature of the SystemCore Driver Station?
The new Driver Station adds rumble and LED output, 16-bit axes, and support for up to 64 buttons and 8 POVs. It keeps full gamepad support, with much more of it.
