Variables, Types & Math

Numbers the
robot remembers.

A variable is a labeled box that holds a value. Get the handful of types and one division gotcha, and you can compute anything a robot needs.

Code is mostly nouns and arithmetic.

A variable stores a value under a name so you can use it later: speed = 2.4 makes a box called speed holding 2.4. In Python you don't write the type in front - the value itself decides what kind it is. Even so, it helps to picture four kinds of value you'll use constantly:

int

A whole number: 5, -12, 0. CAN ids, counts, button numbers.

float

A decimal number: 2.4, -0.5, 3.14159. Almost everything physical - speeds, voltages, positions.

bool

True or false: True, False. Is the button pressed? Are we at the target?

str

Text: "Elevator". Mostly for names and dashboard labels.

The classic trap

    
// throws away the decimal

    
One gotcha bites every beginner. Python has two division operators. Plain / always keeps the decimal, so 5 / 2 is 2.5. But // is floor division - it drops the fractional part, so 5 // 2 is 2, not 2.5. Reach for // only when you actually want a whole number.

    

      
The trap, and the fix Python


a, b = 5, 2

wrong = a // b      # 2    <- floor division drops the decimal!
right = a / b       # 2.5  <- plain / keeps it
also  = a / 2.0     # 2.5  <- a decimal works too

    

    

      
Where it bites a robot

      
Gear-ratio and unit math is full of division. ticks // counts_per_rev silently rounds to a whole number, and your mechanism thinks it's at the wrong place. When a value can have a decimal, divide with /, not //.

Compute the average of two integer sensor readings without losing the decimal. Fill the blank operator so avg is 2.5 when the readings are 2 and 3.

Going further · optional

      
If you already code, predict each result before you run it: 0.1 + 0.2 == 0.3 is actually False, because a float stores numbers in binary and cannot hold 0.1 exactly, so tiny rounding creeps in. This is why robot code compares decimals with a tolerance (is it within a small epsilon?) instead of asking for an exact match. Look up "floating point rounding" and try printing 0.1 + 0.2 to see the leftover digits.

Question 1

        
Which type would you use for a motor's speed in meters per second?

        

          
          
          
        

        
Speeds, voltages, and positions are rarely whole numbers, so they're floats. int is for whole-number things like CAN ids and counts.

Question 3

        
What does a variable's type tell you?

        

          
          
          
        

        
The type (int, float, bool, str…) is what sort of value lives in that variable - and it changes how operations like division behave.
