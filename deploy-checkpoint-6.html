Question 1
A driver pushes a single stick fully forward and halfway right, and the code calls arcadeDrive(fwd, turn). What does the left side get compared to the right side?
Arcade mixing is left = fwd + turn, right = fwd - turn. A forward-and-right command speeds up the left wheels and slows the right, curving the robot to the right while it still drives ahead. Review: Lesson 6.1 · The Drivetrain
Question 2
Your swerve robot needs to slide straight down-field while its body stays turned 30 degrees to one side. Which capability makes that possible?
Swerve is holonomic because every module steers, so the robot can translate any direction while pointed any way. fromFieldRelativeSpeeds rotates the stick command by the gyro heading so push-away always means down-field. Review: Lesson 6.2 · Swerve Drive in Code
Question 3
During a long match the robot's odometry estimate of its position slowly slides away from where it really is. Why does this happen?
Odometry is a running sum of motion from a known start. Each tiny measurement error, such as a slipping wheel, is added to the total and never corrected, so the estimate drifts. A vision fix or a known reset re-anchors it. Review: Lesson 6.3 · Odometry
Question 4
A PathPlanner path looks great in the GUI, but the robot keeps cutting the corner during autonomous. Which thing most directly determines how tightly it hugs the planned line?
A path follower is a control loop on your pose: each loop it asks where it should be versus where odometry says it is, then closes the gap with PID and feedforward. A drifting pose means a wrong error, so it cuts the line. Better odometry means tighter tracking. Review: Lesson 6.4 · Trajectories & PathPlanner
Question 5
You want the shooter spinning up while the robot is still driving to the goal during an autonomous routine. How do you express that, given the command pieces from Part 3?
An auto is a command composition. sequence runs steps in order; alongWith runs them in parallel. Overlapping a spin-up with a drive means the shooter is at speed the moment the robot arrives, which shaves time off the routine. Review: Lesson 6.5 · Building an Autonomous
