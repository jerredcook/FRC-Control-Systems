Question 1
You are building an arm. Which split between a subsystem and a command is correct?
A subsystem is the noun: it owns the hardware and updates itself in periodic() every 20 ms. A command is the verb: it asks the subsystem to do something. Only the subsystem touches its own motor, which stops two pieces of code from fighting over it.
Question 2
A command is running and partway through, another command that requires the same subsystem gets scheduled. What happens, and where should the first command shut its motor off?
The scheduler lets only one command own a subsystem at a time. Scheduling a new one interrupts the old, and end(interrupted) always fires, even on interrupt. That is why stopping the motor and other cleanup belong in end(), not isFinished().
Question 3
In autonomous you want to run an intake but give up if it has not grabbed a piece after 3 seconds, so the routine never hangs. Which composition fits?
withTimeout races the command against a timer and ends when either finishes, so the intake stops after at most 3 seconds. andThen would run them one after another, and a parallel group waits for all members, so neither caps the time.
Question 4
You bind driver.a().whileTrue(shootCommand) and give the shooter a default command. What is the behavior?
whileTrue runs the command while the button is held and ends it on release. A default command is the fallback a subsystem runs when nothing else requires it, so releasing A interrupts shootCommand and the default resumes on its own.
Question 5
Back in Part 2 you learned a Java method can return a boolean. A Trigger reacts to any boolean condition, not only buttons. Which use of that is correct?
A Trigger wraps any boolean supplier, including a method like hasGamePiece() that returns true or false. onTrue schedules a command once when the condition flips to true, so the robot reacts to a sensor automatically, the same way it reacts to a button.
