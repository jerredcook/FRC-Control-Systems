Question 1
You unplug your laptop's USB cable right after a successful deploy, then enable teleop. What happens to the robot?
Deploy copies your built program onto the roboRIO, and from then on it runs there on its own. The laptop is only needed to write, build, and watch telemetry, so disconnecting it does not stop the robot.
Question 2
Your robot is connected and the Robot Code light is green, but every motor reads zero output and nothing moves. The Driver Station says Disabled. Why?
In Disabled the program keeps running and the periodic methods keep firing, but WPILib forces motor outputs to zero so nothing moves. You enable a mode like Teleop in the Driver Station before the robot can drive anything.
Question 3
A teammate flashed new firmware on your Kraken motors, then your build started throwing protocol errors that were not there yesterday. What is the most likely fix?
A vendordep is a JSON file in vendordeps/ that pulls in the vendor library at build time. Its version must match the firmware on the devices. Krakens use CTRE Phoenix 6, so update the Phoenix6 vendordep to match and commit it so the whole team stays in sync.
Question 4
You finished a working intake feature on a branch. What is the safe way to get it into the shared main branch with the rest of the team?
A pull request proposes merging your branch and gives teammates a chance to review before it reaches main. Pull before you push so you build on the team's latest, and never force push over shared history.
Question 5
Which sequence describes the everyday robot programming loop, and where does each step happen?
You write and build on the laptop, deploy the compiled program to the roboRIO, then enable and observe through the Driver Station before going again. Most of the skill is making that loop fast so you can try more ideas before a match.
