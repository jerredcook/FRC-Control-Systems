Question 1
You write double ratio = 60 / 9; expecting about 6.67, but printing it shows 6.0. What went wrong?
When both sides of a division are int, Java does integer division and drops the decimal, giving 6. The result is only converted to double afterward. Write 60.0 / 9 or 60 / 9.0 to keep the fraction.
Question 2
You want a value that can only ever be one of STOWED, INTAKING, or SCORING, and the compiler should reject anything else. What fits best?
An enum is a named, fixed set of values. The compiler only allows those three, which prevents invalid states and reads better than 0, 1, 2. A switch on the enum then forms a clean state machine.
Question 3
Your robot has four swerve modules. You build them in a constructor and store them so each keeps its own angle and speed. Which is the most fitting structure?
Each module is its own object made with new, so each holds independent state. An array is the right fit for a known, fixed count like four modules. Reusing one object would make all modules share the same fields, which is wrong.
Question 4
You write a method deadband(double value) that returns 0 when the input is small and otherwise returns the value. You call it from several places. What does the return value let you do?
A method takes parameters as input and hands back a return value as output. Because the cleanup lives in one named method, every caller gets the same behavior and you fix bugs in one spot instead of many copies.
Question 5
In Part 1 you saw Robot extends TimedRobot and methods marked @Override. Now that you know Java, what does @Override actually buy you on something like robotPeriodic()?
extends gives Robot all of TimedRobot's machinery for free, and @Override marks a method as replacing the parent's version. If you misspell robotPeriodic, the compiler catches it instead of quietly creating a new method that never gets called every 20 ms.
