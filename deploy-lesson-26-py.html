In code

Design in a GUI, follow in one line

You draw paths in PathPlanner or Choreo and save them by name. Your code loads a path and wraps it in a follow command, ready to drop into an autonomous routine:

Following a planned path Python · PathPlanner
# load a path you drew in the PathPlanner GUI
path = PathPlannerPath.fromPathFile("ScorePickup")

# a command that follows it using your pose + drive controllers
follow = AutoBuilder.followPath(path)

# or load a whole multi-path routine by name
auto = PathPlannerAuto("FourPiece")

# you configure AutoBuilder once with your drive, kinematics, and PID
The follower is a control loop on your pose

Following isn't dead reckoning (blindly running a fixed set of moves and hoping it lines up). Each loop the follower asks "where should I be on this path right now?" and "where does odometry say I am?", then applies feedback (the PID and feedforward from the Control course, Part 5) to close the gap. That's why accurate odometry matters so much: the better the robot knows its pose, the tighter it hugs the planned line.

Going further · optional

A real gotcha: PathPlanner paths are saved relative to the blue alliance origin, so a path that scores perfectly from the blue side will drive off the field if you run the same code on red without flipping it. AutoBuilder takes a "should flip" supplier so one path works for both alliances by mirroring across the field. Try sketching it: if the field is a mirror down the middle, what happens to an x-coordinate and to a heading when you flip a path to the other alliance?

Show the thinking

Mirroring across the center of the field's length flips x to (field length minus x), so a point near the blue wall lands the same distance from the red wall, while y stays the same. A heading of theta becomes (180 degrees minus theta), since the direction is reflected left-to-right: straight downfield at 0 degrees flips to 180 degrees, and a heading already pointing across the field is barely changed. PathPlanner's "should flip" supplier does this math for you, so you draw the path once for blue and let it mirror for red.

Quick check

Pick an answer to see why.

Question 1
What is a trajectory (path)?
A trajectory is a continuous curve fit through waypoints, carrying a position and velocity for every moment in time. The robot follows it during autonomous instead of being driven by hand.
Question 2
How does the path follower stay on the planned line?
Every loop the follower computes the error between where it should be and where odometry says it is, then applies the same PID and feedforward from the Control course to drive that error to zero.
Question 3
What are PathPlanner and Choreo?
They're path-design tools: you place waypoints visually, they generate a smooth, feasible trajectory, and you load it by name. Choreo optimizes for the fastest path; PathPlanner is waypoint-based with events.
