Part 5 · 04 The Drivetrain
Swerve Modules

Two motors
per wheel.

Swerve is the most capable drivetrain in FRC, and it works by giving every wheel two motors: one to drive it and one to steer it. Each wheel can point any direction and spin any speed, so the robot can move sideways, diagonally, and spin all at once.

One module: a wheel that points and drives.

A swerve module is a single wheel with a steer motor that rotates the whole assembly to any heading, and a drive motor that spins the wheel. Set the steering angle and drive speed below and watch one module respond.

The steer motor swings the module to whatever heading you set; the drive motor spins the wheel at whatever speed you ask. Four of these on a robot, each pointing where it needs to, is what lets swerve glide in any direction while it spins.

This is where build hands off to code

The module is the hardware; figuring out what angle and speed each of the four should be at is the software. That's exactly the swerve kinematics from the Deploy course (Lesson 6.2): you tell the code "drive this direction and spin this fast," and it computes the four module states. Mechanically you buy and bolt on proven modules (the buy-not-build call from Lesson 2.3); in code you command them. Swerve is the clearest place the two halves of this academy meet.

Going further · optional

Here is a subtlety the demo hides: a steer motor only ever has to turn a half-circle, never a full one. If the code asks a wheel to point backward, it is faster to flip the wheel to the near heading and spin the drive motor in reverse instead, so the module never rotates more than 90 degrees to reach any direction. Teams also trade off gearing: a faster drive ratio means a higher top speed but less push, the same speed-versus-force tradeoff from the gearing lessons, now decided per module. Why does spinning the drive backward let the steering take a shortcut?

Show the thinking

A wheel is symmetric front to back, so pointing it at a heading and driving forward moves the robot the same way as pointing it 180 degrees opposite and driving in reverse. That gives the steering two equivalent ways to reach any target, and they are always a half-turn apart. The code picks whichever one is closer and flips the drive direction if it chose the far side, so the module never has to rotate more than 90 degrees to reach any commanded direction. That makes steering moves quicker and avoids the wheel chasing the long way around.

Checkpoint

Quick check

Pick an answer to see why.

Question 1
What two motors does a swerve module have?
Each module has a drive motor (wheel speed) and a steer motor (heading). Together they let one wheel point any direction and drive any speed.
Question 2
What does a four-module swerve let a robot do?
With every wheel independently steered, the robot is holonomic: it can translate any direction and spin simultaneously, far more agile than a tank drive.
Question 3
Should you build swerve modules from scratch or buy them?
Modules are complex and precise, so nearly every team buys proven COTS modules. The code (swerve kinematics) is what you write to control them.
