Build, Deploy, Run

Compile your project, send it across, and enable it from the Driver Station - closing the loop from Lesson 1 for real.

Three verbs turn your folder of files into a moving robot.

Build compiles your Java into a program. Deploy copies that program onto the roboRIO and starts it. Then the Driver Station - the app that talks to the robot - enables it, so motors are allowed to move. Try the whole thing below: deploy the code, pick a mode, and enable.

You can't enable until code is deployed and the robot reports "Robot Code" - the Driver Station won't let motors move otherwise. That's a safety feature, not a bug.

The four modes

What "enabled" means

An FRC robot is always in one of a few states, and the Driver Station picks which. While disabled, your code still runs - but motor outputs are forced to zero, so nothing moves. Enable a mode and the matching code takes over:

Disabled · Teleoperated · Autonomous · Test

Teleoperated - you drive with the controller. Autonomous - the robot runs a pre-programmed routine on its own. Test - for checking mechanisms in the pit. Disabled - code runs, motors don't. In a real match the field controls these for you; on the bench, the Driver Station does.

The space bar is E-STOP

In the Driver Station, the spacebar (and the big red button on a real one) is an emergency stop - it kills the robot instantly. Know where it is before you ever enable a robot near people.

Going further · optional

A full deploy rebuilds and resends everything, which is slow when you only tweaked one number. Checkpoint

Quick check

Pick an answer to see why.

Question 1
What does "deploy" do?
Build compiles; deploy sends the compiled program to the roboRIO and launches it. After a successful deploy the robot reports "Robot Code" and is ready to be enabled.
Question 2
Your code is deployed and running, but the robot is disabled. What happens?
Disabled doesn't stop your program - periodic methods still run. It just blocks motor output for safety. Nothing moves until you enable a mode.
Question 3
Which mode runs a pre-programmed routine with no driver input?
Autonomous runs your autonomousPeriodic / auto command on its own - the first ~15 seconds of an FRC match. Teleoperated is driver control; Test is for pit checks.
