Configuring a TalonFX

Set it up
once.

Current limits, brake mode, sensor ratios - a one-time setup before the motor can safely run your loop. Build one below and watch the Python write itself.

Configuration is just a settings object.

Every TalonFX setting lives in a TalonFXConfiguration - one object with nested groups: motor_output (brake/coast, direction), current_limits (protect the motor and breakers), feedback (the sensor ratio from Lesson II.2), and later slot0 and motion_magic. You fill in the fields, then call apply() once at startup, and the motor remembers.

What each group does

    
The settings that matter most

    

      
Current limits
The most important safety setting. A stator limit caps torque and protects the motor from burning up; a supply limit keeps you from tripping a breaker. Always set them.

      
Neutral mode
Brake resists motion when idle (an arm holds its spot); Coast lets it spin free (a drivetrain rolls). Pick per mechanism.

      
sensor_to_mechanism_ratio
The Lesson II.2 gear ratio, set once - now get_position() returns mechanism rotations and the onboard loop tunes in mechanism units.

      
FusedCANcoder
Combines an absolute CANcoder (knows its angle at power-on) with the fast internal rotor - best of both. Essential for arms and swerve steering.

    

    

      
Why "Fused"

      
A bare CANcoder is absolute but updates slowly; the internal rotor is fast but forgets its zero at power-off. FusedCANcoder uses the CANcoder for the true angle and the rotor for speed and smoothness - so an arm knows exactly where it is the instant the robot boots, and still runs a crisp 1 kHz loop.

    

  


  

    
Applying it

    
One call at startup

    
In the subsystem constructor, build the config and apply it. apply() sends every setting to the device and waits for confirmation - do it once, not every loop:

    

      
elevator.py - constructor Python · Phoenix 6


def __init__(self):
    super().__init__()
    self.motor = TalonFX(10)

    cfg = TalonFXConfiguration()
    cfg.motor_output.neutral_mode = NeutralModeValue.BRAKE
    cfg.current_limits.stator_current_limit = 80
    cfg.current_limits.stator_current_limit_enable = True
    cfg.feedback.sensor_to_mechanism_ratio = 9.0
    self.motor.configurator.apply(cfg)     # once - the device remembers

Going further · optional

      
The stator limit caps current right at the windings, but it also caps torque, since torque rises with current, like how hard you can push the pedals sets how fast you can climb a hill. Set it too low and your arm cannot lift its own weight; too high and you cook the motor or pop a breaker. A real edge case competitive teams hit: apply() returns a status code, and if a device is missing or miswired the config silently never lands. The careful move is to check that code, or call apply() in a small retry loop. What would a robot do if its current limits never actually applied?

    

    
Checkpoint

    
Quick check

    
Pick an answer to see why.

    

      

        
Question 1

        
Which setting most directly protects your motor from burning up?

        

          
          
          
        

        
The stator current limit caps the current actually flowing through the motor windings - that's what generates heat and torque. A supply limit additionally protects your breaker. Always set both.

      

      

        
Question 2

        
For an arm that must hold its position when the robot is disabled, which neutral mode do you want?

        

          
          
          
        

        
Brake mode shorts the motor leads when idle, resisting motion, so the arm doesn't flop down. Coast is for things you want to roll free, like a drivetrain you push around the pit.

      

      

        
Question 3

        
What does a FusedCANcoder give you that the internal rotor alone can't?

        

          
          
          
        

        
The internal rotor is fast but only counts from wherever it booted. A CANcoder is absolute - it knows the true angle immediately. Fusing them gives an arm or swerve module instant, accurate position plus a smooth high-rate loop.
